# Winget Package Manifests

These manifests define the Toorker package for Windows Package Manager (winget).

## Structure

- `1etu.Toorker.yaml` - Default locale manifest
- `1etu.Toorker.installer.yaml` - Installer configuration
- `1etu.Toorker.locale.en-US.yaml` - English locale

## Testing Locally

Before submitting to winget-pkgs, test locally:

```powershell
# Install from local manifests
winget install --manifest .\.winget\1etu.Toorker.yaml

# Or validate the manifest
winget validate .\.winget\
```

## Publishing to Winget

### Prerequisites

1. Have a published GitHub Release with installers
2. Fork https://github.com/microsoft/winget-pkgs
3. Install winget-create: `winget install wingetcreate`

### Automated (Recommended)

```powershell
# Generate manifests automatically
wingetcreate new --urls https://github.com/1etu/toorker/releases/download/v0.1.0/Toorker_0.1.0_x64-setup.exe --version 0.1.0
```

### Manual Submission

1. Copy manifests to: `winget-pkgs/manifests/1/1etu/Toorker/0.1.0/`
2. Create a pull request to microsoft/winget-pkgs
3. Wait for validation and approval (usually 24-48 hours)

### Updating

For new releases:

```powershell
wingetcreate update 1etu.Toorker --urls https://github.com/1etu/toorker/releases/download/v0.2.0/Toorker_0.2.0_x64-setup.exe --version 0.2.0
```

## Automation via GitHub Actions

The repository includes a workflow (`.github/workflows/winget.yml`) that automatically submits to winget-pkgs on new releases.

**Setup required:**
1. Create a personal access token with `public_repo` scope
2. Add as `WINGET_TOKEN` secret in your repository
3. The workflow will run on new releases and create a PR to winget-pkgs
